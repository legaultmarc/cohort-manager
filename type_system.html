Type system¶

An important feature of this project is the unified representation of variables across data type. This is accomplished using a hierarchical type system where every data type defines its own validation function.

Tree representing the relationship between data types. Most of the time, all the validation checks of the parent types are applied when checking a subtype.

The types.Type class¶

Parent type for all types.

static check(values)[source]¶

Check if a data vector is of the appropriate type.

static decode(hdf5_dataset)[source]¶

Method that is used to restore data from HDF5.

If a special encoding method was provided, implementing this method is essential.

static encode(values)[source]¶

Method that gets called when storing the data in the HDF5 dataset.

This can be overriden by children who need a special encoding step.

Note

This function gets called automatically and can be used to encode arrays that are not handled correctly by H5PY. It should not be used to process information.
